## Teach Hydra to DO Things

Skills make Hydra knowledgeable. Integrations make Hydra connected. Actions make Hydra **capable**. Drop an `action.toml` into the `actions/` folder.

```
actions/
  deploy/
    action.toml     <- what to execute, when, with what approval
```

## action.toml Format

<Tabs>
  <Tab title="Manual">
    ```toml
    [action]
    name        = "deploy"
    description = "Deploy the application to production"
    trigger     = "manual"
    approval    = "required"

    [action.execute]
    type            = "shell"
    command         = "cd /path/to/project && git pull && docker-compose up -d"
    timeout_seconds = 300

    [[action.parameters]]
    name     = "environment"
    type     = "string"
    default  = "production"
    ```
  </Tab>
  <Tab title="Scheduled">
    ```toml
    [action]
    name        = "backup-database"
    description = "Backup the production database to S3"
    trigger     = "scheduled"
    schedule    = "every day at 3:00 AM"
    approval    = "notify"

    [action.execute]
    type            = "shell"
    command         = "pg_dump $DATABASE_URL | gzip | aws s3 cp - s3://backups/db-$(date +%Y%m%d).sql.gz"
    timeout_seconds = 600
    ```
  </Tab>
  <Tab title="Conditional">
    ```toml
    [action]
    name        = "alert-high-cpu"
    description = "Alert when CPU usage exceeds 90%"
    trigger     = "conditional"
    approval    = "auto"

    [action.condition]
    check_command  = "top -l 1 | grep 'CPU usage' | awk '{print $3}'"
    threshold      = 90
    comparison     = "greater_than"
    check_interval = "every 5 minutes"
    ```
  </Tab>
</Tabs>

## Three Approval Modes

| Mode | Behavior | Use For |
|------|----------|---------|
| `required` | Hydra asks "Approve?" and waits for confirmation | Spending money, sending messages, modifying systems |
| `auto` | Hydra executes without asking | Safe read-only actions, notifications, internal tasks |
| `notify` | Hydra executes and tells you afterwards | Scheduled tasks, monitoring, backups |

## Execution Types

Actions can execute shell commands or API calls:

```toml
# Shell execution
[action.execute]
type    = "shell"
command = "cargo test && docker-compose up -d"

# API execution
[action.execute]
type   = "api"
method = "POST"
url    = "https://api.example.com/endpoint"
body   = '{"key": "{value}"}'
```
